# Спецификация доменной модели бронирования

## Контекст

Приложение позволяет гостям бронировать встречи в календаре одного владельца. В проекте есть две условные роли: владелец календаря и гости.

Регистрация и авторизация отсутствуют. Владелец календаря считается заранее заданным системным профилем и используется по умолчанию в административной части. Гость бронирует слоты без создания аккаунта и без входа в систему.

## Владелец календаря

Владелец календаря - единственный заранее заданный профиль, которому принадлежат все типы событий и все бронирования.

Поля:

- `id` - идентификатор владельца.
- `name` - имя владельца, отображаемое в интерфейсе.
- `timezone` - часовой пояс календаря.

Правила:

- В системе существует только один владелец календаря.
- Владелец не проходит регистрацию и не выполняет вход.
- Административная часть всегда работает от имени этого владельца.

## Тип события

Тип события описывает вид встречи, которую может выбрать гость.

Поля:

- `id` - уникальный идентификатор типа события.
- `title` - название события.
- `description` - описание события.
- `durationMinutes` - длительность встречи в минутах.
- `ownerId` - идентификатор владельца календаря.

Правила:

- Тип события создается владельцем календаря.
- Тип события доступен гостю на публичной странице видов брони.
- Длительность типа события используется при построении доступных слотов и создании бронирования.

## Слот

Слот - конкретное доступное время начала встречи для выбранного типа события.

Поля:

- `startAt` - дата и время начала слота.
- `endAt` - дата и время окончания слота.
- `eventTypeId` - тип события, для которого рассчитан слот.
- `durationMinutes` - длительность слота, берется из типа события.
- `isAvailable` - признак доступности слота.

Правила:

- Слоты формируются динамически на ближайшие 14 дней, начиная с текущей даты.
- Слот доступен только если на это же время нет другого бронирования.
- Нельзя создать две записи на одно и то же время, даже если они относятся к разным типам событий.
- Гость может выбрать только свободный слот из 14-дневного окна записи.

## Бронирование

Бронирование - подтвержденная запись гостя на конкретное время.

Поля:

- `id` - уникальный идентификатор бронирования.
- `eventTypeId` - выбранный тип события.
- `ownerId` - владелец календаря.
- `guestName` - имя гостя.
- `guestContact` - контакт гостя, например email или телефон.
- `startAt` - дата и время начала встречи.
- `endAt` - дата и время окончания встречи.
- `createdAt` - дата и время создания бронирования.

Правила:

- Бронирование создается гостем без регистрации и входа.
- Время окончания рассчитывается как `startAt + durationMinutes` выбранного типа события.
- Перед созданием бронирования система проверяет, что выбранный слот свободен.
- Если на выбранное время уже существует бронирование, новое бронирование не создается.
- Бронирование должно попадать в окно записи: от текущей даты до текущей даты плюс 14 дней.
- Владелец видит все предстоящие бронирования в одном списке, независимо от типа события.

## Публичный сценарий гостя

1. Гость открывает публичную страницу видов брони.
2. Система показывает список доступных типов событий.
3. Для каждого типа события отображаются название, описание и длительность.
4. Гость выбирает нужный тип события.
5. Система открывает календарь со свободными слотами на ближайшие 14 дней.
6. Гость выбирает свободный слот.
7. Гость вводит данные, необходимые для бронирования.
8. Система повторно проверяет доступность выбранного времени.
9. Если слот свободен, система создает бронирование.
10. Если слот уже занят, система сообщает гостю, что время больше недоступно, и предлагает выбрать другой слот.

## Ключевые ограничения

- В приложении нет пользователей, регистрации и авторизации.
- Владелец календаря один и задан заранее.
- Гости не имеют аккаунтов.
- Тип события влияет на длительность встречи, но не позволяет обойти занятость календаря.
- В один и тот же момент времени может существовать только одно бронирование.
- Запись возможна только в пределах ближайших 14 дней.
