# E2E-тесты

Тесты используют `pytest` и `playwright`. Фикстуры в `conftest.py` перед запуском сценариев поднимают backend и frontend на свободных локальных портах, поэтому тесты не зависят от занятых портов `3000` и `5173`.

## Команды

```bash
make tests-install
make tests-e2e
```

## Сценарии

### `test_guest_can_create_booking`

Проверяет публичный сценарий гостевого бронирования:

- открывает главную страницу;
- выбирает тип события `Знакомство`;
- выбирает первый свободный слот;
- заполняет имя и контакт гостя;
- подтверждает бронирование;
- проверяет уведомление `Бронирование создано`.

### `test_owner_can_create_event_type`

Проверяет сценарий владельца календаря:

- открывает кабинет владельца `/owner`;
- проверяет, что отображается владелец `Александр Пушкин`;
- переходит во вкладку `Типы событий`;
- открывает форму создания типа события;
- заполняет название, описание и длительность;
- создает новый тип события;
- проверяет уведомление `Тип события создан` и появление нового типа в списке.
